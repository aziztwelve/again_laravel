# Задача: Функционал «Скоро в продаже»

**Статус:** Спека утверждена — готова к реализации
**Дата:** 2026-06-23
**Раздел сайта:** Каталог → категория «Скоро в продаже»
**Раздел админки:** Каталог товаров → Товары онлайн; новая категория

### Принятые решения
- **Форма (#1):** выезжающая справа панель «Узнать о поступлении» (drawer, как мини-
  корзина). Поля по порядку: `имя` (опц.), `телефон` (опц., маска +7, → WhatsApp),
  `email` (**обязателен**). Кнопка «Подписаться» неактивна до согласия. **Структура —
  по скриншотам ТЗ, стили/компоненты — из дизайн-системы сайта** (раздел «Вёрстка»).
- **Категория (#2): вариант A** — флаг `categories.is_coming_soon`, авто-наполнение
  товарами `is_active = true AND stock_quantity = 0` (минуя pivot, как `is_new_product`).
- **Уровень подписки (#3):** на **товар целиком**. `product_variant_id` сохраняем
  для информации (если клиент смотрел конкретный вариант), но триггер — на уровне
  `Product.stock_quantity` (0 → >0).
- **Каналы гостя (#4):** email + WhatsApp (по `phone`). Telegram/VK — только
  авторизованным клиентам с привязкой (`telegram_user_id` / `vk_user_id`).
- **Дедуп (#5):** дубль = `(product_id, email)` среди `status = pending`. Повторная
  заявка — идемпотентно (success, новую строку не создаём).
- **Повторные циклы (#6):** подписка **одноразовая**. После уведомления → `notified`
  (терминальный статус). Повторное поступление не шлёт повторно; клиент подписывается заново.
- **Письмо (#7):** заголовок «УЖЕ В НАЛИЧИИ», текст «"{Название}" уже на сайте.
  Успейте заказать нужный размер», блок товара в 2 колонки (фото | название + цена +
  кнопка «КУПИТЬ» → `/catalog/{slug}`). Тема письма «Уже в наличии — Again».
  **Структура — по скриншоту ТЗ, стили — в фирменных цветах** (Blade, inline-CSS,
  кнопка `#CB0B13`; см. раздел «Вёрстка»).
- **Согласие ПДн (#8):** обязательный чекбокс согласия в форме (ссылки на
  `personal-data-consent` / `marketing-consent`).
- **Отписка (#9):** в первой версии **не делаем** (подписка одноразовая, гаснет
  после уведомления).
- **Фронты (#10):** витрина — `again_front`, заявки в админке — `again_dashboard`.

## Описание

Добавить функционал «Скоро в продаже»: возможность публиковать на сайте карточки
товаров, которых **ещё нет в наличии**, и собирать заявки «Сообщить о поступлении».
Когда товар появляется в наличии онлайн — автоматически уведомлять подписавшихся
клиентов по email и в мессенджеры со ссылкой на карточку товара.

**План-результат:**
- В каталоге появляется категория «Скоро в продаже».
- В неё добавляются товары, которых нет в наличии, но карточка тянется из МойСклад.
- Карточка товара без наличия отображается **полностью** (как обычная), но вместо
  кнопки «В корзину» — кнопка «Сообщить о поступлении».
- По кнопке открывается форма; после отправки — уведомление об успешной заявке.
- При появлении товара в наличии онлайн — рассылка подписчикам (email + мессенджеры)
  с кнопкой «Перейти к товару» (ссылка на карточку на сайте).

> Скриншоты формы и письма из ТЗ — на закрытом CDN (подписанные токены), не открывались.
> Состав полей формы и текст письма зафиксированы в «Принятых решениях» (#1, #7)
> и при наличии доступа к макетам сверяются перед версткой.

## Сценарий

### Менеджер (админка)
1. Открывает «Каталог товаров» (товары, синхронизированные из МойСклад).
2. Находит товар без остатка и публикует его в «Товары онлайн» (`is_active = true`).
3. Привязывает товар к категории «Скоро в продаже» (или товар попадает в неё
   автоматически по флагу, см. решение #2).
4. Видит счётчик заявок «Сообщить о поступлении» по товару.

### Покупатель (сайт)
1. Заходит в каталог → «Скоро в продаже» (или открывает карточку товара без наличия).
2. Видит полноценную карточку товара; вместо «В корзину» — кнопка «Сообщить о
   поступлении» (с иконкой-колокольчиком).
3. Нажимает кнопку → справа выезжает панель «Узнать о поступлении» с мини-карточкой
   товара (бейдж «Нет в наличии») и полями: имя, телефон, эл. почта + чекбокс согласия.
4. Нажимает «Подписаться» → видит подтверждение «Мы сообщим вам о поступлении».
5. Когда товар появляется в наличии онлайн → получает письмо/сообщение в мессенджер
   со ссылкой «Перейти к товару».

## Модель данных (предлагаемая)

### Категория «Скоро в продаже»
**Решение #2 — вариант A (по флагу).** Добавить `categories.is_coming_soon` (boolean,
default false) по аналогии с `is_new_product`. В `CatalogService` для такой категории
показывать товары `is_active = true AND stock_quantity = 0` (минуя pivot-привязку).
Товар автоматически уходит из категории, как только появляется остаток.

> Отклонённая альтернатива (вариант B): обычная категория + ручная привязка через
> `category_product` — потребовала бы ручной отвязки при поступлении. Не используем.

## Триггер «товар появился в наличии онлайн»

Условие отправки: товар стал доступен к покупке, т.е. переход
**`stock_quantity` из 0 в > 0** при `is_active = true`.

**Подход: `ProductObserver`** (триггер на уровне товара, решение #3).
- В `updated()` проверять `isDirty('stock_quantity')` и переход
  `getOriginal('stock_quantity') <= 0 && stock_quantity > 0`.
- Если есть `pending`-подписки на этот товар и `is_active = true` —
  диспатчить `NotifyRestockSubscribersJob` (очередь).
- Observer покрывает **все** пути изменения остатка (синк МойСклад, ручная правка,
  списания), а не только синхронизацию. Регистрация — в `AppServiceProvider`.
- Примечание: при синке остаток товара пересчитывается как сумма остатков вариантов
  (`syncVariantsForProduct` → `$product->update(['stock_quantity' => $totalStock])`),
  поэтому хук на `Product` ловит и появление любого варианта.

> Отклонённая альтернатива: ловить переход прямо в
> `ProductsAndVariantsSyncWithMoySkladService`. Минус — не покрывает ручные
> изменения остатка. Observer надёжнее.

**Job `NotifyRestockSubscribersJob`:**
1. Берёт `pending`-подписки по `product_id`.
2. Для каждой формирует ссылку на карточку: `FRONTEND_URL . '/catalog/' . $product->slug`
   (роут карточки на фронте — `again_front/pages/catalog/[slug].vue`).
3. Шлёт уведомление по доступным каналам (`SendNotificationJob`), решение #4:
   - **email** — всегда (email обязателен), письмо «УЖЕ В НАЛИЧИИ» с кнопкой «КУПИТЬ» (шаблон ниже);
   - **whatsapp** — если есть `phone` (через `PhoneFormatterTrait::formatPhoneForWhatsApp`);
   - **telegram / vk** — только если подписка привязана к `client` с
     `telegram_user_id` / `vk_user_id`.
4. Помечает подписку `status = notified`, `notified_at = now()`.
5. **Идемпотентность (#6):** подписка одноразовая. Шлём только `pending`; после
   рассылки → `notified` (терминальный). Повторное поступление не уведомляет повторно —
   клиент подписывается заново.

### Шаблон письма о поступлении (#7) — по скриншоту ТЗ
- **Тема:** «Уже в наличии — Again».
- **Тело (структура из макета):** логотип Again → заголовок «УЖЕ В НАЛИЧИИ» →
  текст «"{name}" уже на сайте. Успейте заказать нужный размер» → блок товара в
  2 колонки: фото | название (uppercase) + цена + **кнопка «КУПИТЬ»** (ссылка на
  карточку, фирменный цвет `#CB0B13`).
- В мессенджерах — текстовый аналог со ссылкой (как `BirthdayDiscountCommand`),
  напр.: «"{name}" уже в наличии! Успейте заказать: {ссылка}».
- Полная вёрстка письма — в разделе «Вёрстка».

## Backend: API (предлагаемые эндпоинты)

### Публичные (витрина)
- `POST /restock-subscriptions` — создать подписку.
  - Body: `product_id` (required), `product_variant_id` (nullable),
    `name` (nullable), `email` (**required**, email), `phone` (nullable),
    `consent` (**required**, accepted — согласие ПДн, #8), `meta` (nullable).
  - Валидация: товар существует, `is_active = true`, `stock_quantity = 0`
    (подписка имеет смысл только для отсутствующего товара).
  - Анти-спам: rate-limit по IP, дедуп по `(product_id, email)` среди `pending`
    (повтор — идемпотентно success).
  - Ответ: `201` + сообщение для тоста «Мы сообщим вам о поступлении».

- Карточка/список товара (существующие публичные эндпоинты) должны отдавать товар
  **даже при `stock_quantity = 0`** (если `is_active = true`) и возвращать в ресурсе:
  - `in_stock: bool` (`stock_quantity > 0`);
  - `cta: 'add_to_cart' | 'notify_restock'` (что показывать на карточке);
  - на уровне варианта — `in_stock` для каждого варианта (см. #3).

### Категория «Скоро в продаже»
- Отдаётся существующим эндпоинтом меню каталога / списка категорий.
- При выбранной категории «Скоро в продаже» `CatalogService` фильтрует
  `is_active = true AND stock_quantity = 0` (вариант A).

### Админка
- `GET /admin/restock-subscriptions` — список заявок (фильтр по товару/статусу), пагинация.
- `GET /admin/products/{id}/restock-subscriptions/count` — счётчик заявок по товару
  (или отдавать `restock_subscriptions_count` в ресурсе товара админки).
- Управление категорией «Скоро в продаже» — существующий CRUD категорий
  (+ переключатель `is_coming_soon`, если вариант A).

## Вёрстка: форма «Узнать о поступлении» и письмо

> Принцип (по требованию): **структура — как на скриншотах ТЗ; стили, компоненты,
> шрифты и цвета — из существующей дизайн-системы сайта** (`again_front`).
> Новых стилей не вводим — переиспользуем готовые классы и компоненты.
> На скриншотах ТЗ бренд-референс `Cerca Trova` (зелёный акцент); берём **структуру**,
> оформляем в стиле Again (акцент — красный `#CB0B13`).

### Форма — выезжающая справа панель (drawer), НЕ центрированная модалка
По скриншоту форма открывается как **slide-over панель справа**, как мини-корзина.
→ Опора: механизм бокового drawer `useAsideMenuStore()` + `components/AsideMenu.vue`
(`_aside-menu.scss`), как у `MiniCart/Block.vue`. Новый компонент содержимого —
`components/Catalog/RestockNotify.vue` (или `components/Modals/...`, но именно в drawer).

**Структура панели (точно по скриншоту):**
1. **Заголовок** «Узнать о поступлении» + крестик закрытия (справа сверху).
2. **Подзаголовок** «Мы уведомим вас, когда товар появится в наличии».
3. **Мини-карточка товара** (в светлом скруглённом блоке): миниатюра + название +
   красный бейдж «Нет в наличии».
4. **Поле** «Ваше имя».
5. **Поле телефона** с маской `+7 (___) ___-__-__`.
6. **Поле** «Эл. почта».
7. **Чекбокс согласия** (один): «Я соглашаюсь на получение информационных и рекламных
   сообщений от Again и подтверждаю ознакомление с электронным согласием на рассылку»
   (ссылка подчёркнута).
8. **Кнопка** «Подписаться» — на всю ширину, внизу панели, **неактивна до согласия**.

## Этапы реализации (черновой план)

1. Миграция + модель `product_restock_subscriptions` (+ индексы, дедуп `(product_id, email)`).
2. Миграция `categories.is_coming_soon` + логика выборки в `CatalogService`.
3. Публичный `POST /restock-subscriptions` + FormRequest-валидация (email + consent) + rate-limit.
4. Ресурсы карточки/списка: отдавать товар при `stock_quantity = 0`, добавить
   `in_stock` и `cta`; правка `ProductDetailResource` / `ProductResource`.
5. `ProductObserver`: детект перехода остатка 0 → >0, диспатч
   `NotifyRestockSubscribersJob` (регистрация observer в `AppServiceProvider`).
6. `NotifyRestockSubscribersJob`: выборка `pending`, рассылка по каналам
   (`SendNotificationJob`), пометка `notified`, идемпотентность.
7. Письмо `resources/views/emails/product-restock.blade.php` (inline-CSS, фирменные
   цвета): заголовок «УЖЕ В НАЛИЧИИ», блок товара в 2 колонки, кнопка «КУПИТЬ» →
   `/catalog/{slug}` (база — `order-receipt.blade.php`).
8. Фронт `again_front`: drawer-форма `Catalog/RestockNotify.vue` (паттерн `MiniCart`/
   `useAsideMenuStore`), `Product/Actions/NotifyRestock.vue` (кнопка с колокольчиком),
   свап CTA по `in_stock`/`cta`, тост/`ModalsSuccess`.
9. Фронт `again_dashboard`: список заявок, счётчик, переключатель категории.
10. Тесты: подписка (валидация/дедуп/спам), триггер перехода остатка,
    рассылка по каналам, идемпотентность.

## Безопасность / граничные случаи

- **Публичный эндпоинт подписки** без аутентификации → rate-limit по IP, дедуп
  по `(product_id, email)`, валидация email/телефона. Обязательный чекбокс согласия
  ПДн/маркетинг (#8; ссылки на `marketing-consent`, `personal-data-consent` во фронте).
- **Дубли подписок (#5):** один и тот же email на один товар среди `pending` — не
  плодить (идемпотентный success).
- **Гонка при синке:** массовый синк МойСклад может одновременно «открыть» много
  товаров → рассылку выполнять в очереди, `withoutOverlapping`, батчами.
- **Повторные циклы (#6):** подписка одноразовая — после уведомления `notified`
  (терминальный). Повторное поступление не уведомляет повторно; клиент подписывается заново.
- **Вариативные товары (#3):** наличие на витрине считается по варианту, но триггер
  и подписка — на уровне `Product.stock_quantity` (при синке = сумма остатков вариантов).
  `product_variant_id` хранится информативно.
- **Категория «Скоро в продаже» и сортировка:** текущая сортировка каталога ставит
  товары с остатком вперёд (`CASE WHEN stock_quantity > 0`). Для этой категории все
  товары с нулевым остатком — сортировку пересмотреть (например, по `display_order`/дате).
- **Канал для гостя (#4):** у гостевой заявки — email (обяз.) + WhatsApp (по phone).
  Telegram/VK — только привязанному `client`.
- **Отписка (#9):** в первой версии не делаем (подписка одноразовая).
